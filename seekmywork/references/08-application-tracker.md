# 投递状态追踪

## Overview

追踪已投递岗位的状态，记录面试进度，管理求职节奏。

## When to Use

- ✅ 用户开始投递后，想记录投递状态
- ✅ 用户说"帮我记录一下""我投了 XX 公司"
- ✅ 用户想回顾投递情况、统计进度

## Core Process

### Phase 1 — 初始化

检查是否存在追踪文件 `投递追踪_{日期}.md`：
- 不存在 → 创建初始文件
- 已存在 → 读取现有数据，追加新记录

### Phase 2 — 记录投递

用户告知投递信息时，记录以下字段：

| 字段 | 说明 | 必填 |
|------|------|------|
| 公司 | 公司名称 | ✅ |
| 岗位 | 投递的岗位名称 | ✅ |
| 投递日期 | 投递时间 | ✅ |
| 渠道 | 官网/牛客/内推/招聘会 | 选填 |
| 匹配度 | Module 5 给出的匹配分数 | 自动 |
| 状态 | 待筛选→笔试→一面→二面→HR面→Offer→拒信 | 更新 |
| 备注 | 面试感受、薪资、反馈等 | 更新 |

### Phase 3 — 状态更新

用户可以随时更新状态：

| 用户说 | 操作 |
|--------|------|
| "XX 公司进面试了" | 更新状态为"一面" |
| "XX 公司拒了" | 更新状态为"拒信" |
| "拿到 XX 的 offer" | 更新状态为"Offer" |
| "XX 的面试感觉怎么样" | 追加备注 |

### Phase 4 — 进度汇总

随时生成进度报告：

```markdown
## 📊 求职进度汇总

> 统计时间：{日期}

### 总览
- 总投递：15 家
- 待筛选：5 家
- 面试中：6 家
- 已拿 Offer：2 家
- 已拒：2 家

### 按状态分布

| 状态 | 数量 | 公司 |
|------|------|------|
| 🟡 待筛选 | 5 | 小米、字节、... |
| 🔵 面试中 | 6 | 安克创新、腾讯、... |
| 🟢 Offer | 2 | XX、YY |
| 🔴 拒信 | 2 | ZZ、WW |

### 本周待办
- [ ] 周三：安克创新二面
- [ ] 周五：腾讯笔试
```

### 输出文件格式

```markdown
# 投递追踪

> 创建时间：{日期}
> 最后更新：{日期}

## 投递记录

| # | 公司 | 岗位 | 投递日期 | 渠道 | 匹配度 | 状态 | 备注 |
|---|------|------|---------|------|--------|------|------|
| 1 | 安克创新 | 安全工程师实习 | 06-05 | 官网 | 🟢82% | 一面 | 06-10 面试 |
| 2 | 腾讯 | 后端开发实习 | 06-04 | 内推 | 🟡65% | 待筛选 | |
| ... | | | | | | | |
```

保存为 `投递追踪.md`（持续更新，不按日期分文件）。

## Rules

- ✅ 每次投递都记录，不要遗漏
- ✅ 状态变更时及时更新
- ✅ 匹配度从 Module 5 自动继承
- ✅ 备注如实记录，包括负面反馈

## Verification

- [ ] 追踪文件已创建
- [ ] 投递记录包含必填字段
- [ ] 进度汇总准确反映当前状态
