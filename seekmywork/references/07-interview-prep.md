# 面试准备

## Overview

基于目标岗位 JD 和用户简历，生成面试准备材料：高频问题、回答框架、技术知识点清单、模拟面试题目。

## When to Use

- ✅ 用户已有简历且锁定了目标岗位
- ✅ 用户说"帮我准备面试""面试会问什么"
- ✅ Module 5 匹配完成后，用户想进一步准备
- ✅ 用户拿到了面试邀请，需要针对性准备

## Core Process

### Phase 1 — 前置检查

- [ ] 简历数据存在（`简历_*.md` 或 `简历_*.json`）
- [ ] 目标岗位 JD 存在（`jobs/*.json`）

缺少任一 → 提示用户先完成对应模块。

### Phase 2 — 面试问题预测

根据 JD + 简历，预测面试可能问到的问题，分三类：

#### 2a — 通用行为题（必考）

基于简历经历，预测 HR/技术面都会问的问题：

| 题目类型 | 示例 |
|---------|------|
| 自我介绍 | 请用 1-2 分钟介绍自己 |
| 项目深挖 | 你这个图书管理系统，说说架构设计？ |
| 遇到的困难 | 项目中遇到最大的技术挑战是什么？ |
| 团队协作 | 有没有和队友意见不合的时候？怎么处理的？ |
| 职业规划 | 为什么选择安全方向？未来 3 年规划？ |
| 反问环节 | 你有什么想问我们的？ |

#### 2b — 技术题（根据 JD）

从 JD 中提取技术关键词，生成对应知识点清单：

```
=== 技术知识点清单 ===

岗位：安全工程师实习生

🔴 必备（JD 明确要求）：
  - 渗透测试流程：信息收集 → 漏洞扫描 → 漏洞利用 → 后渗透 → 报告
  - SQL 注入：原理、类型（联合注入/盲注）、防御方案
  - XSS：反射/存储/DOM 型、CSP 绕过
  - 常见工具：Burp Suite、Nmap、SQLMap

🟡 常见（同岗位高频考点）：
  - CSRF / SSRF 原理与防御
  - 密码学基础：对称/非对称加密、哈希
  - 网络协议：TCP 三次握手、HTTPS 握手流程
  - Linux 基础命令

🟢 加分（能答出来加分）：
  - CTF 经验
  - 代码审计
  - 安全应急响应
```

#### 2c — 场景题（根据岗位类型）

| 岗位类型 | 场景题示例 |
|---------|-----------|
| 安全 | 给你一个登录页面，如何测试？ |
| 后端 | 设计一个短链服务，怎么考虑？ |
| 前端 | 页面加载很慢，你怎么排查？ |
| 数据 | 有一份用户行为数据，怎么分析流失原因？ |

### Phase 3 — 回答框架

为每类问题提供 STAR 回答框架：

```
=== 回答模板 ===

项目深挖（以图书管理系统为例）：

S（背景）：课程设计项目，需要实现一个本地图书管理系统
T（任务）：我负责全栈开发，包括后端接口和前端页面
A（行动）：
  - 后端用 Java + JDBC 实现 CRUD
  - 设计了用户表、图书表、借阅记录表三张表
  - 实现了按书名/作者的模糊搜索功能
R（结果）：
  - 支持 100+ 图书的管理
  - 搜索响应时间 < 100ms
  - 课程评分 90+（如果有的话）
```

### Phase 4 — 输出

生成 `面试准备_{公司}_{岗位}_{日期}.md`：

```markdown
# 面试准备：{公司} — {岗位}

> 准备时间：{日期}

## 自我介绍（1 分钟版）

（基于简历自动生成，用户可修改）

## 高频问题 & 建议回答

### Q1：请介绍一下你的项目经历
（STAR 框架回答）

### Q2：为什么选择安全方向？
（建议回答要点）

...

## 技术知识点清单

（按 🔴必备 🟡常见 🟢加分 分级）

## 场景题

（2-3 道岗位相关场景题 + 解题思路）

## 反问建议

1. 这个岗位的日常工作内容是什么？
2. 团队的技术栈是什么？
3. 实习生有转正机会吗？
```

保存为 `面试准备_{公司}_{岗位}_{日期}.md`。

## Rules

- ✅ 所有问题必须基于 JD 和简历生成，不凭空编造
- ✅ 技术知识点必须标注来源（JD 要求/行业常识）
- ✅ 回答框架只提供思路，不写完整话术（避免背稿感）
- ✅ 反问建议必须是真实有价值的问题

## Verification

- [ ] 问题覆盖通用行为题 + 技术题 + 场景题
- [ ] 技术知识点按优先级分级
- [ ] 回答框架使用 STAR 法则
- [ ] 文件已保存且路径已告知用户
