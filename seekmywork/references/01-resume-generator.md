# 简历生成

## Overview

通过对话采集用户的基本信息、技能、实习和项目经历，生成一份结构化 Markdown 简历。也可解析用户已有的简历文件，提取为统一结构。

## When to Use

### 适用场景
- ✅ 用户没有简历，需要从零生成
- ✅ 用户有简历但格式不规范，需要结构化
- ✅ 完整求职流程的第一步

### 不适用场景
- ❌ 用户只需要简历排版建议（不生成文件）
- ❌ 用户简历是扫描件/图片且无法 OCR

## Core Process

### Phase 1: 信息采集

**规则：一次只问一个问题。用户回答后再问下一个。**

按以下顺序采集，每条等用户回复后再问下一条：

**Q1 — 基本信息**
> 请先告诉我你的基本情况：学校、专业、学历（本科/硕士/博士）、毕业时间。

记录：学校名称、专业、学历层次、毕业年月。标注是否 985/211/双一流（如实标注，不评价）。

**Q2 — 技术/技能栈**
> 你会哪些技术/技能？按编程语言、框架工具、其他技能分类列一下。不确定怎么分类的话随便说几个，我来帮你整理。

记录：按「编程语言 / 框架工具 / 其他技能」分类，按用户自述标注熟练度。

**Q3 — 实习经历**
> 有没有实习经历？有的话告诉我：公司名、岗位、做了什么、持续多久。没有也没关系。

记录：公司 + 岗位 + 时长 + 核心产出（一句话概括每条）。无实习则标记「暂无」。

**Q4 — 项目经历**
> 做过哪些项目（课程项目、个人项目、竞赛项目都算）？项目名、用了什么技术、你负责什么、有什么成果/亮点。

记录：项目名 + 技术栈 + 角色 + 亮点。

**Q5 — 其他亮点（可选）**
> 有没有竞赛获奖、论文发表、开源贡献、技术博客、学生干部等经历？没有可以跳过。

**完成条件：** Q1-Q4 全部回答完毕（Q5 可跳过）。向用户复述采集到的信息摘要，请用户确认。

### Phase 2: 简历生成

根据采集信息生成 Markdown 简历，必须包含以下模块：

```markdown
# [姓名] 的简历

## 基本信息
- 学校 | 专业 | 学历 | 预计毕业：YYYY.MM

## 教育背景
学校名称，专业，GPA（如有），相关课程（如有）

## 技术栈
| 分类 | 内容 |
|------|------|
| 编程语言 | ... |
| 框架/工具 | ... |
| 其他技能 | ... |

## 实习经历
### [公司名] — [岗位]（起止时间）
- 用 STAR 法则描述，量化成果

## 项目经历
### [项目名]（时间）
- 技术栈：...
- 个人职责：...
- 项目亮点/成果：...

## 竞赛/荣誉/其他
（如有）
```

### Phase 3: 优化建议

生成简历后，主动给出 2-3 条优化建议：
- 哪些经历可以进一步量化（数字 > 形容词）
- 哪些表述可以更精炼
- 哪些关键词应突出（针对常见校招 JD）

### Phase 4: 保存输出

- 保存为 `简历_{用户名}.md` 到当前工作目录
- 告知用户文件路径
- 询问是否继续下一步或导出其他格式

### 已有简历的解析

如果用户已有简历文件：
1. 用 `read_file` 读取 .md / .txt 文件
2. .docx 文件：使用 `ocr-and-documents` skill 解析，或请用户转为文本
3. PDF 文件：尝试 `read_file`，失败则请用户提供文本版
4. 解析后输出与 Phase 2 相同格式的结构化简历，让用户确认

## Rules

### 必须遵守
- ✅ 一次只问一个问题，不要列出清单轰炸用户
- ✅ 用户确认信息摘要后再生成简历，不要跳过确认环节
- ✅ 用数据量化成果（"提升 30%" > "显著提升"）
- ✅ 保留用户原始表述，不要擅自改写或缩小范围
- ✅ 对学校/学历如实标注，不做价值评判

### 禁止事项
- ❌ 编造用户的实习/项目经历
- ❌ 对学校做"好/差"的评价
- ❌ 一次性列出所有问题
- ❌ 跳过确认环节直接生成

## Common Rationalizations

| 借口 | 现实 | 应对 |
|------|------|------|
| "用户信息太少，生成不了" | 信息少也可以生成基础版 | 生成后标注"建议补充"项，引导用户完善 |
| "简历内容差不多，复用上次的" | 每个用户都是独特的 | 严格按采集到的信息生成 |
| "用户说跳过就跳过吧" | 跳过确认可能导致错误 | 至少让用户看一遍摘要再继续 |

## Red Flags

- 🚩 用户说"随便帮我写一个"：必须基于真实信息，不能编造
- 🚩 简历文件是扫描件且无法提取文本：告知用户局限，请提供文本版
- 🚩 用户要求写虚假经历：明确拒绝

## Examples

### 示例 1：从零生成简历

**输入：**
> 用户：中南大学，计算机科学与技术，本科，2026年毕业。会 Java、Python、MySQL。在字节跳动做过3个月后端实习。做过一个图书管理系统。

**输出：**
生成 `简历_张三.md`，包含：
- 基本信息：中南大学 | 计算机科学与技术 | 本科 | 2026.06
- 技术栈：Java(熟练)、Python(熟练)、MySQL(熟练)
- 实习：字节跳动 — 后端开发实习生（3个月）
- 项目：图书管理系统（Spring Boot + MySQL）

## Verification

完成后确认：
- [ ] 简历包含所有必要模块（基本信息、教育、技能、经历）
- [ ] 所有经历基于用户真实提供的信息
- [ ] 用户已确认信息摘要无误
- [ ] 简历文件已保存且路径已告知用户
