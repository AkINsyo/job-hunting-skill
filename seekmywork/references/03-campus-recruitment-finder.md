# 企业校招页面查询

## Overview

查找指定企业的校园招聘官网入口，优先匹配已知域名库，未命中则搜索验证，最终输出可访问的 URL 汇总。

## When to Use

### 适用场景
- ✅ 用户想找某公司的校招网站
- ✅ 用户想批量获取多家公司的校招入口
- ✅ Module 3 执行前需要校招页面 URL

### 不适用场景
- ❌ 用户想找具体岗位（→ Module 3）
- ❌ 用户想找社招页面（本模块聚焦校招）

## Core Process

### Phase 1: 解析输入

从用户输入中提取公司名列表：
- 逗号分隔：`小米,字节跳动,腾讯`
- 空格分隔：`小米 字节跳动 腾讯`
- 自然语言：「帮我找一下小米和字节的校招网站」

**未指定公司时**，使用默认列表：
```
小米, 字节跳动, 腾讯, 阿里巴巴, 百度, 华为,
美团, 京东, 网易, 快手, 小红书, 滴滴,
哔哩哔哩, 拼多多, 蚂蚁集团, 联想, 中兴, OPPO, vivo
```

### Phase 3: 搜索校招页面

对未命中的公司，用 `web_search` 按优先级搜索：

1. `{公司名} 校招官网 2025 2026`
2. `{公司名} campus recruitment official`
3. `{公司名} 校园招聘`

**提取规则**：
- 优先提取官方域名（如 `hr.xiaomi.com`）
- 排除第三方平台（牛客、知乎、CSDN）
- 含 `campus`、`recruit`、`join`、`career` 路径的优先

**并行搜索**：使用 `delegate_task` 的 tasks 批量模式，每次最多 3 家并行。

### Phase 4: 验证可访问性

对每个 URL 执行 curl 验证：

```bash
curl -sL -o /dev/null -w "%{http_code}" --max-time 10 "URL"
```

- `200` → ✅ 可访问
- `301/302` → ⚠️ 重定向（记录跳转目标）
- 其他 → ❌ 不可访问

### Phase 5: 输出

生成汇总表格并保存为 `校招网站汇总.md`：

```markdown
# 🎓 校招官网汇总

> 查询时间：{日期}

| 公司 | 校招官网 | 状态 |
|------|---------|------|
| 小米 | https://hr.xiaomi.com/campus | ✅ |
| ... | ... | ... |
```

## Rules

### 必须遵守
- ✅ 优先使用已知域名库，减少不必要的搜索
- ✅ 搜索结果必须附原始链接，不可编造
- ✅ 对不可访问的链接如实标记，不可隐瞒

### 禁止事项
- ❌ 编造校招 URL
- ❌ 将第三方平台（牛客、知乎）当作官方校招入口
- ❌ 跳过验证直接输出

## Common Rationalizations

| 借口 | 现实 | 应对 |
|------|------|------|
| "域名看起来对，不用验证" | 域名可能已变更或失效 | 必须 curl 验证 |
| "搜索结果第一条就是" | 搜索结果可能被广告/SEO 干扰 | 核对是否为官方域名 |
| "已知域名库肯定对" | 企业可能更换校招入口 | 每次仍需验证可访问性 |

## Red Flags

- 🚩 搜索结果全是第三方平台：扩大搜索关键词
- 🚩 curl 返回 403 但浏览器可访问：标注"可能有反爬限制，建议手动访问"
- 🚩 公司名太小众搜不到：告知用户，建议直接搜索公司官网找招聘入口
