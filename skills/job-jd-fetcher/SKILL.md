# 企业招聘岗位查询 & JD 拉取

## Overview

从企业校招页面或招聘平台搜索具体岗位，拉取 JD（职位描述）并提取为结构化数据，保存为 JSON + Markdown 双格式到本地。

## When to Use

### 适用场景
- ✅ 用户想看某公司在招哪些岗位
- ✅ 用户想拉取特定方向的岗位 JD
- ✅ Module 4 需要岗位数据作为输入

### 不适用场景
- ❌ 用户只需要校招网站链接（→ Module 2）
- ❌ 用户想找社招岗位（本模块聚焦校招/实习）

## Core Process

### Phase 1: 获取输入

需要以下信息（从用户获取或从上游模块继承）：

| 信息 | 必填 | 来源 |
|------|------|------|
| 公司名 | ✅ | 用户指定 |
| 岗位方向 | 选填 | 用户指定（如"后端开发"） |
| 城市 | 选填 | 用户指定，默认不限 |

### Phase 2: 获取校招页面 URL

- 如果 Module 2 已执行且输出了该公司 URL → 直接使用
- 否则在已知域名库中查找 → 匹配则使用
- 都没有 → 调用 Module 2 Phase 3 的搜索逻辑获取

### Phase 3: 搜索岗位

对每个公司，用 `web_search` 执行多轮搜索：

**搜索策略（按优先级）：**

1. **精准搜索**：`{公司名} {岗位方向} 校招 2025 2026`
2. **平台搜索**：`site:nowcoder.com {公司名} {岗位方向} 校招`
3. **广域搜索**：`{公司名} 校招 岗位 招聘 2025`

**执行规则**：
- 每个查询 topK 设为 5-8
- 对搜索结果中有详细 JD 的链接，用 `web_fetch` 抓取具体内容
- 每轮搜索后挑 2-3 个最有价值的链接深度打开
- 覆盖至少 3 个不同查询词
- **所有结果必须来自真实网页，不得编造**

### Phase 4: 提取结构化 JD

从每个岗位页面中提取：

```json
{
  "company": "公司全称",
  "position": "岗位名称",
  "city": "工作地点",
  "education": "学历要求",
  "skills": ["技能1", "技能2"],
  "responsibilities": ["职责1", "职责2"],
  "bonuses": ["加分项1", "加分项2"],
  "deadline": "截止日期",
  "source_url": "原始链接"
}
```

### Phase 5: 保存到本地

**目录结构**：
```
jobs/
  {公司名}_岗位.json     ← 结构化数据
  {公司名}_岗位.md       ← 人类可读版
```

**JSON 格式**：
```json
{
  "company": "小米",
  "fetched_at": "2026-06-05",
  "positions": [
    {
      "position": "后端开发工程师",
      "city": "北京",
      "education": "本科及以上",
      "skills": ["Java", "MySQL", "Redis"],
      "responsibilities": ["负责核心服务开发"],
      "bonuses": ["有大厂实习经验"],
      "deadline": "2026-07-31",
      "source_url": "https://..."
    }
  ]
}
```

**Markdown 格式**：
```markdown
# 小米 — 校招岗位汇总

> 拉取时间：2026-06-05

## 1. 后端开发工程师
- **城市**：北京
- **学历**：本科及以上
- **技能要求**：Java/Go、MySQL、Redis
- **职责**：...
- **加分项**：有大厂实习经验
- **截止**：2026-07-31
- **链接**：[查看详情](URL)
```

### Phase 6: 输出汇总

向用户展示：
- 共拉取了多少个岗位
- 按方向分类统计
- 文件保存位置

## Rules

### 必须遵守
- ✅ 所有岗位必须附原始链接（source_url）
- ✅ 保存 JSON + Markdown 双格式
- ✅ 每个岗位的 skills 字段从 JD 原文提取，不要泛化
- ✅ 多家公司时，为每家生成独立文件

### 禁止事项
- ❌ 编造任何岗位信息
- ❌ 搜索不足时凑数填入不相关岗位
- ❌ 省略 source_url 字段

## Common Rationalizations

| 借口 | 现实 | 应对 |
|------|------|------|
| "搜到的信息差不多了" | 差不多 ≠ 充分 | 至少覆盖 3 个查询词 |
| "JD 太长，摘要一下就行" | 摘要会丢失细节 | 保留完整 skills 和 responsibilities |
| "这个链接打不开，跳过吧" | 可能是临时问题 | 重试一次，仍失败则标注 |

## Red Flags

- 🚩 搜索结果全是第三方转载：尝试直接访问校招官网
- 🚩 JD 内容与岗位名称不符：可能是挂羊头卖狗肉，标注后告知用户
- 🚩 用户指定的公司搜不到校招信息：如实告知，建议确认公司是否有校招

- [ ] JSON 文件格式正确可解析
- [ ] Markdown 文件人类可读
- [ ] 文件已保存到 jobs/ 目录
