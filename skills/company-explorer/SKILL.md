# 公司遍历与筛选

## Overview

根据用户的理想行业、专业背景（来自简历）和目标城市，遍历筛选出合适的目标公司；再通过网络调研这些公司的员工评价（工作环境、薪资福利、发展前景等）；最终综合多项维度输出一份目标公司推荐清单。

**核心价值：** 在用户盲目投递之前，先帮他锁定一批"值得投"的公司，避免海投浪费时间。

## When to Use

### 适用场景
- ✅ Module 1（简历生成）完成后，用户需要确定目标公司
- ✅ 用户说"帮我找找长沙有哪些好公司"
- ✅ 用户想了解某行业在某城市的公司分布
- ✅ 用户想对比多家公司的口碑

### 不适用场景
- ❌ 用户已明确指定目标公司（直接跳到 Module 3 校招查询）
- ❌ 用户只需要校招网站链接（→ Module 3）
- ❌ 用户没有简历数据（提示先执行 Module 1）

## Core Process

### Phase 1: 采集筛选条件

从用户输入或上游模块继承以下信息：

| 信息 | 必填 | 来源 |
|------|------|------|
| 目标行业 | ✅ | 用户指定（如"人工智能""网络安全""互联网"） |
| 专业背景 | ✅ | Module 1 简历数据（如"计算机科学""信息安全"） |
| 目标城市 | ✅ | 用户指定（如"长沙""深圳"） |
| 目标公司 | 选填 | 用户指定（如有则优先，不做遍历） |
| 公司规模偏好 | 选填 | 大厂/中厂/创业公司/不限 |

**信息采集规则：**
- 一次只问一个问题
- 如果用户在 Module 1 已提供部分信息，直接继承，不重复询问
- 如果用户说"不限"，按行业主流公司覆盖

### Phase 2: 公司遍历

#### 2.1 已知公司库匹配

首先在本地已知公司库中查找匹配的公司（见下方知识库）。

#### 2.2 网络搜索补充

对未覆盖的公司，使用 `web_search` 搜索：

**搜索策略：**
1. **行业 + 城市搜索**：`{城市} {行业} 公司 排名`
2. **产业园区搜索**：`{城市} {行业} 产业园 企业`
3. **招聘平台搜索**：`{城市} {行业} 招聘 公司`
4. **行业报告搜索**：`{行业} {城市} 企业名录 2025`

**并行搜索**：使用 `delegate_task` 的 tasks 批量模式，每批最多 3 个查询。

**提取规则**：
- 优先提取本土总部公司（HQ 在目标城市）
- 其次提取在目标城市有研发中心的大厂
- 排除与用户专业完全不匹配的公司
- 每家公司记录：公司名、行业、规模、总部/分支、主营业务

#### 2.3 初步筛选

根据以下条件过滤：

| 筛选维度 | 规则 |
|---------|------|
| 行业匹配 | 公司主营业务与用户目标行业相关 |
| 专业匹配 | 公司技术栈与用户专业背景有交集 |
| 城市匹配 | 公司在目标城市有办公点 |
| 规模偏好 | 符合用户对大厂/中厂/创业的偏好（如有） |

### Phase 3: 员工评价调研

对筛选后的每家公司（Top 10-15 家），调研员工评价。

#### 3.1 评价数据源

使用 `web_search` 搜索以下平台的员工评价：

1. **看准网**：`site:kanzhun.com {公司名} 怎么样`
2. **脉脉**：`site:maimai.cn {公司名} 评价`
3. **知乎**：`site:zhihu.com {公司名} 工作 体验`
4. **牛客**：`site:nowcoder.com {公司名} 面经 评价`
5. **小红书**：`{公司名} 工作体验 评价`

#### 3.2 评价维度提取

从搜索结果中提取以下维度（如实记录，不做主观评判）：

| 维度 | 说明 | 数据来源 |
|------|------|---------|
| 综合评分 | 看准网评分（如有） | 看准网 |
| 工作氛围 | 员工对团队/文化的描述 | 脉脉、知乎 |
| 薪资福利 | 薪资范围、福利待遇 | 看准网、脉脉 |
| 加班情况 | 是否常态加班、工作时长 | 脉脉、知乎、小红书 |
| 晋升发展 | 晋升通道、成长空间 | 知乎、脉脉 |
| 面试难度 | 面试流程、难度评价 | 牛客、看准网 |
| 正面评价 | 员工提到的优点 | 综合 |
| 负面评价 | 员工提到的不足 | 综合 |

#### 3.3 评价可信度标注

- ⭐⭐⭐⭐⭐ 多平台一致、评价数量多
- ⭐⭐⭐⭐ 评价较多但有分歧
- ⭐⭐⭐ 评价较少
- ⭐⭐ 极少评价或信息矛盾
- ⭐ 无法获取评价

### Phase 4: 综合评分

对每家公司从以下维度打分（1-5 分）：

| 维度 | 权重 | 评分依据 |
|------|------|---------|
| 行业匹配度 | 25% | 公司主营业务与目标行业的匹配程度 |
| 专业匹配度 | 25% | 公司技术栈与用户专业的匹配程度 |
| 员工口碑 | 20% | 员工评价综合得分 |
| 发展前景 | 15% | 公司规模、融资阶段、行业地位 |
| 薪资竞争力 | 15% | 薪资水平（如有数据） |

**综合得分** = 各维度加权得分

### Phase 5: 输出目标公司清单

#### 5.1 汇总表格

```markdown
# 🎯 目标公司推荐清单

> 行业：{目标行业}
> 专业：{用户专业}
> 城市：{目标城市}
> 调研时间：{日期}

| 排名 | 公司 | 行业 | 类型 | 综合评分 | 员工口碑 | 关键优势 | 关键风险 |
|------|------|------|------|---------|---------|---------|---------|
| 1 | XX公司 | AI | 本土上市 | ⭐⭐⭐⭐ | 4.2/5 | 技术氛围好 | 加班较多 |
| 2 | ... | ... | ... | ... | ... | ... | ... |
```

#### 5.2 详细分析（Top 5）

对排名前 5 的公司展开详细分析：

```markdown
## 🏆 Top 1：XX公司

### 基本信息
- **行业**：人工智能
- **规模**：1000-5000 人
- **总部**：长沙
- **上市**：深交所 300XXX

### 员工评价摘要
- **综合评分**：4.2/5（看准网，500+ 评价）
- **工作氛围**：技术导向，团队协作好
- **薪资福利**：中上水平，五险一金齐全
- **加班情况**：项目期较忙，平时正常
- **晋升发展**：有明确晋升通道

### 正面评价
- 技术栈新，能学到东西
- 团队氛围好，领导靠谱

### 负面评价
- 项目赶进度时加班多
- 部分部门管理混乱

### 推荐理由
- 与你的专业高度匹配
- 长沙本土公司，稳定性好
- 员工口碑整体正面
```

#### 5.3 投递建议

```markdown
## 📮 投递建议

### 第一梯队（强烈推荐）
1. XX公司 — 专业最匹配
2. YY公司 — 口碑最好

### 第二梯队（值得尝试）
3. ZZ公司 — 发展前景好
4. ...

### 第三梯队（保底选择）
5. ...

### 建议
- 第一梯队优先投递，同步准备面试
- 第二梯队作为补充选择
- 第三梯队作为保底
```

保存为 `目标公司清单_{日期}.md`。

## Rules

### 必须遵守
- ✅ 公司遍历必须覆盖至少 3 个搜索查询词
- ✅ 员工评价必须注明来源平台，不可编造
- ✅ 评价中正面和负面都要如实记录，不可只报喜
- ✅ 综合评分必须有明确的维度和权重
- ✅ 推荐清单必须有分梯队的投递建议

### 禁止事项
- ❌ 编造员工评价或薪资数据
- ❌ 对公司做"好/坏"的价值评判，只呈现事实
- ❌ 推荐与用户专业完全不匹配的公司
- ❌ 忽略用户指定的目标公司（如有指定，必须纳入）

## Common Rationalizations

| 借口 | 现实 | 应对 |
|------|------|------|
| "这家公司在目标城市没有岗位" | 没有岗位也要告知用户 | 在清单中标注"暂无校招岗位" |
| "员工评价都是负面的" | 负面评价也是有价值的信息 | 如实记录，让用户自己判断 |
| "搜不到太多信息" | 信息少不代表可以编造 | 标注评价可信度为 ⭐，建议用户自行补充调研 |
| "用户已经指定了公司，跳过调研" | 指定公司也需要了解口碑 | 对指定公司也做评价调研 |

## Red Flags

- 🚩 搜索结果中公司大量裁员/倒闭新闻：在清单中标注风险
- 🚩 员工评价中频繁出现"拖欠工资""强制加班"等关键词：高风险标注
- 🚩 用户目标行业在目标城市几乎没有公司：建议扩大城市范围或调整行业
- 🚩 所有公司评分都很低：建议用户重新评估目标
