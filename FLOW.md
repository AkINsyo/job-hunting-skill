## 三种调用模式

### 模式 A：独立调用

用户只触发单个模块，模块之间无依赖。

```
用户：帮我写简历          → 仅执行 Module 1
用户：帮我找长沙的AI公司   → 仅执行 Module 2
用户：找小米的校招网站     → 仅执行 Module 3
用户：看看字节在招什么     → 仅执行 Module 4
```

### 模式 B：链式调用

按依赖关系依次执行。

```
用户：帮我匹配岗位
  → 检查 → 无简历 → 先执行 Module 1
  → 检查 → 无目标公司 → 先执行 Module 2
  → 检查 → 无JD   → 先执行 Module 3 → Module 4
  → 执行 Module 5
  → 询问是否继续 Module 6
```

### 模式 C：完整流程

用户要求从头开始，依次执行全部模块。

```
用户：完整流程 / 帮我从头开始

  Step 1: Module 1 — 采集信息，生成简历
       ↓ 用户确认简历
  Step 2: Module 2 — 根据行业+专业+城市，筛选目标公司
       ↓ 调研员工评价，输出推荐清单
       ↓ 用户确认目标公司列表
  Step 3: Module 3 — 查找目标公司校招网站
       ↓ 用户确认校招入口
  Step 4: Module 4 — 拉取岗位 JD
       ↓ 保存到 jobs/ 目录
  Step 5: Module 5 — 匹配分析
       ↓ 输出匹配结果
  Step 6: Module 6 — 差距分析（询问用户是否需要）
       ↓ 输出提升计划
  完成：告知用户所有产出文件位置
```

## 触发词 → 模块映射

| 用户意图（触发词） | 目标模块 | 说明 |
|-------------------|---------|------|
| 帮我写简历 / 生成简历 / 我没有简历 | Module 1 | 从零生成或解析已有简历 |
| **帮我找公司 / 哪些公司适合我 / 筛选目标公司** | **Module 2** | **行业+专业+城市 → 公司筛选** |
| 找校招网站 / 校招官网 / XX校招在哪 | Module 3 | 查找校招入口 |
| 看看在招什么 / 拉取岗位 / 查职位 | Module 4 | 拉取 JD 到本地 |
| 帮我匹配 / 哪些职位适合我 / 推荐岗位 | Module 5 | 需要 Module 1 + 4 的产出 |
| 分析不足 / 提升建议 / 我还需要学什么 | Module 6 | 需要 Module 1 + 4 的产出 |
| 完整流程 / 从头开始 / 一条龙 | 1→2→3→4→5→6 | 全流程依次执行 |

## Module 2 内部流程（公司遍历与筛选）

```
开始
  ↓
采集筛选条件：目标行业 + 专业 + 城市 + 公司规模偏好
  ↓
已知公司库匹配 → 命中？
  ├── 是 → 纳入候选列表
  └── 否 ↓
网络搜索补充（web_search，3轮不同关键词）
  ↓
初筛：行业匹配 + 专业匹配 + 城市匹配
  ↓
员工评价调研（看准网/脉脉/知乎/牛客/小红书）
  ↓
综合评分：行业匹配25% + 专业匹配25% + 口碑20% + 发展15% + 薪资15%
  ↓
输出分梯队推荐清单（第一/二/三梯队）
  ↓
保存为 目标公司清单_{日期}.md
  ↓
用户确认 → 进入 Module 3
```

## 模块内调用流程

### Module 1 简历生成

```
开始 → Q1基本信息 → Q2技术栈 → Q3实习经历 → Q4项目经历 → Q5其他亮点
  ↓ 复述摘要 → 用户确认 → 生成简历 → 优化建议 → 保存文件
```

### Module 3 校招页面查询

```
开始 → 解析公司列表 → 查询已知域名库 → 搜索校招页面 → curl验证 → 保存汇总
```

### Module 4 岗位 JD 拉取

```
开始 → 获取输入 → 获取校招URL → 搜索岗位 → 提取结构化JD → 保存JSON+MD
```

### Module 5 简历匹配

```
开始 → 前置检查 → 加载数据 → 多维度匹配 → 排序筛选 → 输出结果 → 保存文件
```

### Module 6 差距分析

```
开始 → 前置检查 → 聚合高频要求 → 识别差距 → 分级 → 提升计划 → 简历优化 → 投递策略
```

执行完整流程后，项目目录下会生成：

```
项目目录/
├── 简历_{用户名}.md              ← Module 1
├── 目标公司清单_{日期}.md        ← Module 2（新增）
├── 校招网站汇总.md               ← Module 3
├── jobs/
│   ├── {公司名}_岗位.json        ← Module 4（结构化）
│   ├── {公司名}_岗位.md          ← Module 4（可读版）
│   └── ...
├── 匹配结果_{日期}.md            ← Module 5
└── 提升计划_{日期}.md            ← Module 6
```

## 中途修改偏好

任何时候用户可以修改目标：

| 用户说 | 处理方式 |
|--------|---------|
| "换个行业" | 更新行业，重新执行 Module 2 + 3 + 4 + 5 |
| "城市改一下" | 更新城市，重新执行 Module 2 + 3 + 4 + 5 |
| "加一家公司" | 增量执行 Module 2 + 3 + 4，更新 Module 5 |
| "重新生成简历" | 重新执行 Module 1，然后 Module 2 + 5 + 6 |
| "换一批公司" | 重新执行 Module 2 + 3 + 4 + 5 |

**原则**：只重新执行受影响的模块，不全部重跑。
